Context

Papyrus components offers a maven archetype to generate the basic scafold of a fully compatible Papyrus Component.

How to?

Prerequisite

You need to ask for the creation of a dedicated git repository for your own component.

Create a community ticket based upon this one https://bugs.eclipse.org/bugs/show_bug.cgi?id=496447

Could you please create a new Git repository for the Papyrus project. Name : org.eclipse.papyrus-COMPONENT.git
						Description: Papyrus COMPONENT
						Is it also possible to have the rebase if necessary strategy for the gerrit configuration ?
						Could you wait the +1 of my group leader Remi and the one of the two designer project committers.
						Thanks,

Configuration

You need to have the following repository declare in your .m2 settings.xml file, see https://maven.apache.org/guides/mini/guide-multiple-repositories.html

					<repository>
						<id>pap.repo.eclipse.org</id>
						<name>Papyrus Repository - Releases</name>
						<url>https://repo.eclipse.org/content/repositories/papyrus-releases/</url>
						<releases>
							<enabled>true</enabled>
							<updatePolicy>daily</updatePolicy>
						</releases>
						<snapshots>
							<enabled>false</enabled>
						</snapshots>
					</repository>
					<repository>
						<id>paps.repo.eclipse.org</id>
						<name>Papyrus Repository - Snapshots</name>
						<url>https://repo.eclipse.org/content/repositories/papyrus-snapshots/</url>
						<releases>
							<enabled>false</enabled>
						</releases>
						<snapshots>
							<enabled>true</enabled>
							<updatePolicy>daily</updatePolicy>
						</snapshots>
					</repository>

Generation

Execute the following command to generate the scafold.

				 mvn archetype:generate -DarchetypeGroupId=org.eclipse.papyrus.components -DarchetypeArtifactId=org.eclipse.papyrus.components.archetype -DarchetypeRepository=https://repo.eclipse.org/content/repositories/papyrus-releases/ -DarchetypeVersion=0.0.18-SNAPSHOT -DgroupId=com.cea.PROJECTCATEGORY -DartifactId=com.cea.PROJECTCATEGORY.PROJECT -Dversion=0.0.1 -DfullartifactId=com.cea.PROJECTCATEGORY.PROJECT -DcomponentName=com.cea.PROJECTCATEGORY-PROJECT -DhudsonViewName=HUDSON_TAB -DbugzillaName=BUGZILLA_COMPONENT

Where the following args are defined:

  • Initialize the first group of properties for the components artifact generation: 
    mvn archetype:generate -DarchetypeGroupId=org.eclipse.papyrus.components -DarchetypeArtifactId=org.eclipse.papyrus.components.archetype
  • Register the repository from where to download the version of the archetype: 
    -DarchetypeRepository=https://repo.eclipse.org/content/repositories/papyrus-releases/
  • Describe the version of the archetype you are using (from the nexus repository): 
    -DarchetypeVersion=X.Y.Z
  • The basic triplet for your component. These will identify the group it will contribute to, the name it will be referenced by and the version of the generated component: 
    -DgroupId=org.eclipse.XYZ -DartifactId=org.eclipse.XYZ.componentName -Dversion=X.Y.Z
  • The name of your component and the one that will serve as a basis to build the root package name: 
    -DfullartifactId=org.eclipse.XYZ -DcomponentName=org.eclipse.XYZ.componentName
  • The Hudson tab name: 
    -DhudsonViewName=HUDSON.XYZ
  • The component name in bugzilla: 
    -DbugzillaName=BUGZILLA.XYZ

It will create the following folder tree:

  • ./com.cea.PROJECTCATEGORY.PROJECT: root
  • ./com.cea.PROJECTCATEGORY.PROJECT/src: documentation
  • ./com.cea.PROJECTCATEGORY.PROJECT/com: bug should be deleted
  • ./com.cea.PROJECTCATEGORY.PROJECT/core: a folder to contain all core plugins of your project
  • ./com.cea.PROJECTCATEGORY.PROJECT/releng: the plugins to release your feature, product and rcp
  • ./com.cea.PROJECTCATEGORY.PROJECT/targetplatform: the polugins where you define the target platform against which you want to build your project

You need to check the validty of the generated project by executing a basic mvn clean install

mvn clean install -Dtycho.localArtifacts=ignore

Then push it to your git repository

You will be able then to execute a basic roduct generation

mvn clean install -Pproduct

Hudson

Create the following set of Hudson jobs to ensure the release of your project.

gerrit: to build all changes that comes from a gerrit contribution

gerrit-product: to build all changes coming from gerrit that modified any of the pom.xml or targetplatform files (and could therefore modify the final product

gerrit-website: to build all changes coming from gerrit that modified the site or pom.xml files

XYZ: to build your component based on a specific branch

XYZ-website: to build the component website

XYZ-component-deploy-eclipse: to release your component

XYZ-component-deploy-nexus: to release the nexus artifacts

XYZ-component-deploy-website: to release the website

Have a look to designer component

Create a tab view to group those jobs.
