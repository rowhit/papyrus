54  /**
55   * This class realizes the transformation from component-based to object-oriented
56   * models. It includes the replacement of ports and connectors. Ports are
57   * replaced with attributes and access operations, connectors within a composite
58   * by an operation that creates the initial setup.
59   * 
60   * 1. add an operation that allows to retrieve the reference to an interface provided
61   *    by a port. This operation has a mapping specific name, e.g. get_<port_name>
62   * 2. add an operation that allows to connect a specific port.
63   *    the connect_q operation (*including a
64   *    storage attribute*) for a port with a required interface
65   * 3. add an implementation for the getcnx_q operation for a port
66   *    with a required interface (the operation itself has been added before)
67   * 
68   * Problems: need to align bootloader creation with this mapping, since
69   * the bootloader may be responsible for instantiation
70   * 
71   * Caveat: Assure that the folder derivedInterfaces already exists in a model.
72   * Otherwise the call to getProvided/getRequired interface might trigger its
73   * creation resulting in the corruption of list iterators (ConcurrentAccess
74   * exception)
75   */
76  @SuppressWarnings("all")
